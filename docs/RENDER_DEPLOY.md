# WorkCrew backend deployment on Render

Exact answers for the Render setup form. The backend is the Fastify service in
apps/api. The repository is a monorepo, so the build builds the shared contracts
first, then the API.

## The 13 answers

1. Backend location: apps/api (Fastify, TypeScript, Node).
2. Stripe webhook route already exists: yes.
3. Exact Stripe webhook path: `/v1/billing/webhook`.
4. Render readiness: done (binds to 0.0.0.0 and the Render PORT in production; a render.yaml blueprint is in the repo root).
5. .env.example: present at the repo root with variable names only, no secrets.
6. .env in .gitignore: yes, plus common secret file patterns.
7. Render Build Command: `npm install && npm run build -w @workcrew/contracts && npm run build -w @workcrew/api`
8. Render Start Command: `node apps/api/dist/server.js`
9. Render Root Directory: leave blank (the repo root).
10. Runtime: Node (no Python or Docker needed for the backend; the Python helper runs on the user PC, not the server).
11. Environment variables to paste in Render: see the list below.
12. Port: the server reads the Render PORT automatically and now binds 0.0.0.0 in production. Health check path is `/health`.
13. Final webhook URL: `https://YOUR-RENDER-URL.onrender.com/v1/billing/webhook`

## Environment variables to set in Render

Set by the blueprint automatically:
- NODE_ENV = production
- HOST = 0.0.0.0
- AUTH_MODE = local
- BILLING_MODE = stripe
- WORKCREW_ALLOWED_ORIGINS = app://workcrew
- WORKCREW_LOCAL_AUTH_SECRET = generated by Render

You paste these in the Render dashboard (Environment tab):
- WORKCREW_DATA_URL = your remote libSQL URL (see database note)
- WORKCREW_DATA_AUTH_TOKEN = your remote libSQL token
- ANTHROPIC_API_KEY = your sk-ant- key
- STRIPE_SECRET_KEY = your sk_test_ key
- STRIPE_PRO_MONTHLY_PRICE_ID, STRIPE_PRO_YEARLY_PRICE_ID
- STRIPE_ULTRA_MONTHLY_PRICE_ID, STRIPE_ULTRA_YEARLY_PRICE_ID
- SUPABASE_URL, SUPABASE_ANON_KEY (public values, safe to paste)
- STRIPE_WEBHOOK_SECRET = added after the first deploy, then redeploy

## Database note (important)

The local app uses a SQLite file. Render wipes the local disk on every deploy,
so a file database would lose all accounts and subscriptions. For production use
a remote libSQL database (Turso has a free tier and matches the current code).
Create a Turso database, then set WORKCREW_DATA_URL to its libsql URL and
WORKCREW_DATA_AUTH_TOKEN to its token. No code change is needed.

## Order of operations

1. Push the repo to GitHub (private).
2. In Render, New, Blueprint, select the repo. It reads render.yaml.
3. Paste the secret env vars (all except STRIPE_WEBHOOK_SECRET).
4. Deploy. Copy the Render URL.
5. In Stripe (test mode), add a webhook endpoint at the URL plus
   /v1/billing/webhook, selecting checkout.session.completed, the four
   customer.subscription events, and the two invoice events.
6. Copy the whsec_ signing secret, paste it into Render as
   STRIPE_WEBHOOK_SECRET, and redeploy.
7. Point the desktop app at the Render URL (Settings, once that screen is built).
